ls -> lista arquivos
cd -> move entre diretório
pwd -> mostra diretório atual
mkdir -> cria uma pasta
cat -> abre o conteúdo do arquivo
git init -> defini a pasta especial(cria repositório) na pasta atual
ls -a -> mostra pastas e pastas ocultas
git status -> mostrar como está o diretório
git add nome_arquivo -> prepara um arquivo para publicação de versão
git add . -> prepara todos os arquivos da pasta
git commit -m "documentação versionamento" -> publicar a versão
git config --global user.email -> saber qual usuário está configurado no git
git config --global user.email "vinihada2001@gmail.com" -> definir o e-mail no git
git config --global user.name "viniciusakio2001" -> definir o nome no git
nano nome_arquivo -> cria um arquivo onde poderemos escrever dentro do terminal
git log -> consultar todos os commits
git diff nomearquivo -> para saber o  que foi alterado no estado atual e no estado anterior
git reset nomearquivo -> remove o arquivo do stage(unstage) para não ser comitado
git checkout -b nomebranch -> cria uma branch nova copiando a branch do estado atual e move o estado para a branch criada
git branch -> lista as branchs do projeto e destaca a branch do estado atual
git checkout nomebranch -> move o estado da branch para a branch mencionada
git merge nomebranch(origem) -> transfere os novos arquivos e novas alterações do branch de origem para o branch do estado atual
git branch -d nomebranch -> deleta a branch mencionada
git remote add origin urlgithub > associa um repositorio local a um repositorio remoto hospedado no github
git push -u origin main -> envia os commits para o repositorio remoto e cria o vinculo com o repositorio remoto
git push origin main -> envia commits para o repositorio remoto do github
